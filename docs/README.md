## 開発の進め方（合意ルール）

1. **ドキュメント先行**: 機能追加・仕様変更の前に該当ドキュメントを更新
2. **Plan Mode を活用**: 大きい変更は plan mode → 承認 → 実装
3. **MVP優先**: 各フェーズは「動く最小」を確認してから磨く
4. **回帰**: フェーズ完了時に `08-qa-checklist.md` を一周
5. **バランス値はデータで分離**: 数値（GP閾値・確率・価格）は js/data または js/balance.js に集約し、ロジックと分ける

## 用語

- **GP (Growth Points)**: 稲の成長点。クイズ正解で獲得。
- **米粒 / Grain**: ゲーム内通貨 (🍚)。
- **コンボ**: 連続正解数。倍率に反映。
- **SRS**: Spaced Repetition System。簡易Leitnerで実装。
