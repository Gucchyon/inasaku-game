# 00 ビジョン

## ひとこと

> 作物学（特に水稲生産）の英語を、田んぼを育てながら自然と覚える。
> 学術的に正確でありながら、勝手に開きたくなる気持ちよさを持つこと。

## 対象ユーザー

- 主: 開発者本人（作物学を学ぶ／教える文脈の日本人。論文を読むときに英語で詰まりたくない）
- 副: 同じく水稲生産・作物学に関心のある学生・研究者・農業従事者・趣味人

## 設計の3つの柱

### 1. 学術的正確性（Accuracy）
- 語彙の和訳・例文・難易度・カテゴリは、学会紀事の総説や標準教科書を典拠にする
- 各単語に typical usage を載せる（一般的な使い方を一行で示す）
- 用語は分類学的に整理（生育段階／形態／生理／栽培管理／病害虫／土壌・肥料／育種／収穫・調製／気候／一般）
- 紀事や論文を出典として記録できる仕組み（語彙にsource フィールド）

### 2. 楽しさ（Fun）
- **即時報酬**: 正解の瞬間にGP増加・稲のアニメ・米粒ドロップ・コンボ通知
- **進行可視化**: 稲の段階進行が「目に見えて変わる」こと
- **収集**: 30品種＋アチーブメント＋装飾を集めたくなる
- **意外性**: 天候・病害虫イベントによる小さなサプライズ（5問に一回）
- **永続感**: 田んぼ肥沃度・水路・農具など、投資が長く効くアップグレード

### 3. 続けやすさ（Stickiness）
- **デイリーボーナス**: 連続ログインで右肩上がり
- **SRS**: 間違えた語が再出題されるので「覚え切る」体験ができる
- **目標多層化**: 即時(コンボ)/中期(収穫1回)/長期(品種30種コンプ)を同時に走らせる
- **入力モード**: 4択モードで気軽 / 入力モードでガッツリ

## やらないこと（スコープ外）

- リアルタイム対戦・SNS要素
- 実通貨課金
- 多言語UI（英→日 のみ。UIは日本語のみ）
- 農業の労働シミュレーター化（疲労・体力など）
- 複雑な経営（金融・税務・法律）

## 「面白さ」の判定基準

機能を入れるかどうか迷ったら、以下のチェックを通す：

1. □ 学習量を増やすか？（既知単語が増える、忘却を防ぐ）
2. □ 5秒以内に「何が変わったか」が視覚で分かるか？
3. □ 投資（時間/米粒）が無駄にならない設計か？
4. □ 学会紀事の知見と矛盾しないか？
5. □ 子供がやっても大人がやってもどちらも楽しめるか？

3つ以上 ✓ なら追加候補。

## 「正確性」の判定基準

語彙・解説を追加するときの最低条件：

1. □ 出典が確認可能（学会紀事・国立国会図書館・農林水産省・JIS用語など）
2. □ 例文が技術的に正しい（季節・段階・関係性が破綻していない）
3. □ 同じ和訳を持つ別概念と混同しない注釈がある
4. □ カテゴリ分類が他語と整合する
