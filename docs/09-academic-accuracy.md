# 09 学術的正確性 — ゲーム機構の監修計画

## 目的

「学術的に正確」とは単語の和訳だけではなく、**ゲーム内の事象（成長・施肥・病害虫・気象・品種・収量・品質）が現実の作物科学と整合する**ことを意味する。本ドキュメントは、現状ゲームの各機構を学術知見に照らして見直し、修正方針を整理する。

## 現状ゲーム機構の監修対象

### 1. 病害虫イベント（events_table.js）

各イベントが現実の作物病理学と整合しているかを点検する。

| イベントID | ゲーム上の効果 | 現実の知見 | 評価 | 修正案 |
|---|---|---|---|---|
| `blast_disease` (いもち病) | GP×0.5 持続 8問 | 葉いもち→穂いもちで進行。穂いもちは10-50%の収量損失。葉に紡錘形病斑、25-28°C多湿で蔓延 | 効果は概ね妥当だが「葉いもち vs 穂いもち」の段階区別がない | **要拡張**: 葉いもち(苗-分げつ) と 穂いもち(出穂-登熟) を分離、後者は収量直撃 |
| `sheath_blight` (紋枯病) | 収量-20% | 高温多湿、密植、過剰窒素で発生。茎・葉鞘に病斑。倒伏誘発 | 概ね妥当 | 倒伏連動効果を追加検討 |
| `stink_bug` (カメムシ襲来) | 収量-25% | 玄米に黒い斑点（斑点米）→等級下落。収量より品質に影響 | **誤り**: 収量-25%は過大。品質ダウン(検査等級1-2段階下げ)が現実 | **要修正**: yield_penalty を 5% に下げ、別途 quality_penalty を新設して検査等級概念を入れる |
| `planthopper` (ウンカ大発生) | コンボ倍率×0.8 | トビイロウンカ等は萎縮病・縞葉枯病を媒介。直接吸汁で坪枯れ。深刻な場合は数十%の収量損失 | 効果が弱すぎる | **要修正**: 病気媒介効果を加える（次回いもち病/萎縮病の発生確率UP） |
| `weeds` (雑草繁茂) | GP-10/問 3問 | 雑草は栄養・光・水を競合。ノビエは特に水稲と競合大。早期除草が収量-30%を防ぐ | 効果が弱い・短い | **要修正**: 苗-分げつ期に発生時の効果を強化、長期持続に |
| `typhoon` (台風) | 収量-30% | 出穂期の倒伏で収量-20-50%、登熟期は穂発芽・脱粒も。風害は地域性高 | 概ね妥当 | 出穂期と登熟期で効果差別化 |
| `cool_summer` (冷夏) | GP×0.7 5問 | 障害型冷害(穂ばらみ期)で不稔多発、遅延型(登熟期)で登熟不良。1993年大冷害は収量-30% | **要修正**: 段階別効果がない | **要修正**: 穂ばらみ期(stage 2-3)→不稔率増、登熟期(stage 4)→登熟不良 |
| `heat_wave` (高温障害) | 収量-15% | 登熟期高温(>27°C昼夜平均)で**白未熟粒**(乳白・心白・基白・腹白)発生。品質低下が主、収量影響は小 | **誤り**: yield_penaltyではなくquality_penaltyが正しい | **要修正**: 効果を quality_penalty に変更、白未熟粒の概念を導入 |
| `drought` (干ばつ) | 不正解時 -10 GP | 移植後活着不良、分げつ抑制、登熟期は粒充実不足。水稲では灌漑制度ありなら影響限定 | 概ね妥当 | — |
| `heavy_rain` (大雨) | 不正解時 -5 GP | 倒伏誘発、白葉枯病等の発病助長 | 効果が弱い | 連鎖的な病気誘発を追加検討 |
| `sunny_spell` (好天続き) | GP×1.2 3問 | 日射量↑→光合成↑→乾物生産↑→登熟良好 | 妥当 | — |

### 2. 施肥アイテム（items.js）

| アイテム | ゲーム上の効果 | 現実 | 評価 |
|---|---|---|---|
| `fertilizer_basic` 基本肥料 | GP×1.5 / 10問 | 元肥として代表的肥料。窒素・リン酸・カリの混合 | 妥当 |
| `fertilizer_quick` 速効肥料 | GP×2.0 / 5問 | 硝酸態窒素や尿素など、即効性。追肥に使用 | 妥当 |
| `fertilizer_premium` 高級肥料 | GP×1.7 / 20問 | 緩効性・肥効調節型肥料。長期効果 | 概ね妥当（持続が長いのは正しい） |
| `fungicide_basic` 殺菌剤 | いもち病・紋枯病解除 | トリシクラゾール、フェリムゾン等が代表 | 妥当 |
| `insecticide_basic` 殺虫剤 | カメムシ・ウンカ解除 | 系統別（ネオニコチノイド、有機リン系等） | 簡略化されているが許容 |

**追加検討**：
- 元肥(basal)と追肥(topdressing)の概念分離
- 穂肥(panicle dressing)の追加：出穂20-25日前の窒素追加で穎花数増・登熟向上
- 実肥(grain dressing)：出穂期の窒素で粒重・タンパク向上（ただし食味は低下）

### 3. 品種（varieties.js）

各品種の `baseYield`, `growthSpeed`, `eventResistance` を、現実の品種特性と照合する。

| 品種 | 現状ゲームスペック | 現実の特性 | 評価 |
|---|---|---|---|
| コシヒカリ | yield 80, speed 1.0, 耐性 0% | 良食味だが**いもち病弱・倒伏弱**。中生 | **要修正**: 耐性を負値（-0.05〜-0.10）に |
| あきたこまち | yield 85, speed 1.05, 耐性 5% | 中生。いもち病やや弱、耐倒伏中 | 妥当（耐性は -2%程度が現実的） |
| ササニシキ | yield 95, speed 0.95, 耐性 -5% | 良食味、**冷害に弱い**、いもち病やや弱 | 妥当（冷害脆弱性を反映） |
| 山田錦 | yield 130, speed 0.85, 耐性 0% | 酒米、長稈で**倒伏しやすい**、晩生 | 倒伏脆弱性を耐性負値に反映すべき |
| ゆめぴりか | yield 115, speed 1.0, 耐性 10% | 北海道育成、低アミロース、耐冷性中 | 概ね妥当 |
| つや姫 | yield 110, speed 1.0, 耐性 10% | 山形育成、いもち病やや強、倒伏中 | 妥当 |

**追加検討**：
- 食味スコア（low_amylose, palatability）→ 収穫時の米粒倍率
- 用途別タグ（食用/酒米/もち米/古代米）→ ショップでフィルタ可能に
- 育成系統（コシヒカリ系、ササニシキ系）→ 図鑑の系統樹

### 4. 成長段階（farm.js）

現在: 0種 → 1苗 → 2分げつ → 3出穂 → 4登熟 → 5収穫可

学術的には水稲の成長段階は以下のように分類される（稲学大成より）：
- **栄養成長期**: 発芽→出芽→苗→活着→分げつ盛期→最高分げつ期
- **生殖成長期**: 幼穂形成→穂ばらみ→出穂→開花→受精
- **登熟期**: 乳熟→糊熟→黄熟→完熟

ゲームの6段階は簡略化として妥当だが、以下を追加検討：
- 「活着」概念（移植直後3-7日、新根発生）
- 「最高分げつ期」と「有効分げつ終止期」（1m²あたり穂数を決める）
- 「幼穂形成期」（出穂30日前、肥料反応がもっとも強い）

各段階で発生しやすいイベントの整合：
- 苗→活着不良、立枯病
- 分げつ盛期→雑草競合、分げつ抑制
- 幼穂形成期→冷害(穂ばらみ期不稔)
- 出穂期→台風倒伏、開花期高温不稔
- 登熟期→白未熟粒、胴割粒、刈遅れ

### 5. 気象と影響の整合

| 気象因子 | 現状ゲーム反映 | 現実の影響 |
|---|---|---|
| 気温(積算) | clim_010 で語彙のみ | 出穂後40日積算1000°Cで完熟。登熟温度23°C前後が最適 |
| 日射量 | sunny_spell で表現 | 登熟期日射量↑→千粒重↑、登熟歩合↑ |
| 降水量 | heavy_rain で簡略 | 登熟期多雨→千粒重↓、品質↓ |
| 日長 | 未反映 | 短日処理で出穂前進。日本では晩生品種ほど日長感受性大 |

## 認める簡略化（妥協ライン）

ゲーム性のため、以下は学術正確性より優先：
- 1問=N時間という時間スケールの省略
- 単一作物（水稲）への絞り込み
- 病害虫の薬剤を「殺菌剤」「殺虫剤」に集約（FRACコード等は省略）
- 圃場の地形・水利の地理的差異の省略

これらは「ゲーム説明書」として README またはチュートリアルで明記する。
